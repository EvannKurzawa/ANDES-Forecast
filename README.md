# ANDES-Forecast

**Forecasting the search for biosignatures on habitable rocky worlds with ELT/ANDES**

---

## Pipeline scope and steps

This repository contains the core simulation and Bayesian-detection scripts used to produce the main results of Kurzawa-Ferrandez et al. 2026.

The public pipeline currently includes two main scripts:

### 1. `simulation.py`

This script reads one target planet from the command line, builds a hybrid ANDES S/N grid from ETC v3.2 and v5.2 outputs, generates `petitRADTRANS` transmission spectra for selected molecules, injects the planetary signal into stellar counts, adds noise for several exposure-time assumptions, and exports cross-correlation functions for later Bayesian analysis.

### 2. `bayesian_analysis.py`

This script reads the cross-correlation functions generated by the simulation pipeline, loads the molecule-specific autocorrelation kernels, fits each CCF with both a signal model and a flat/no-signal model using MultiNest, computes Bayes factors, and exports Jeffreys-scale summary files for each planet/exposure/molecule combination.

In the full analysis presented in the paper, these scripts were part of a larger workflow:

- Target selection and ELT visibility screening;
- Barycentric-velocity optimization;
- ETC v3.2 interfacing and ETC v5.2 data retrieval/preprocessing;
- Monte Carlo stability checks;
- Cluster launchers and job-management scripts;
- Final table and figure generation;
- Band-limited comparison runs.

These auxiliary scripts are not included in this repository. They are tightly tied to local data paths, cluster-specific execution, and intermediate analysis choices, and preparing them for a clean public release would require substantial additional restructuring, documentation, and cleanup. We still decided to make the main pipeline publicly available to improve transparency, facilitate access to the methodology, and support reproducibility of the core results.

Some external inputs are therefore expected to be prepared in advance, including the target table passed to `simulation.py`, cached ETC v5.2 outputs, the local ETC v3.2 interface, `petitRADTRANS` opacity files, the required runtime environment, etc.

## Installation and dependencies

This repository is intended for research use and was developed in a local/cluster environment. The scripts are not currently packaged as an installable Python module. A typical setup requires Python 3 and the scientific Python ecosystem, together with several specialized astronomy and Bayesian-inference packages.

The main Python dependencies are:

```bash
numpy
scipy
pandas
matplotlib
tqdm
PyAstronomy
petitRADTRANS
pymultinest
```

Additional non-Python or external requirements include:

- A working installation of **MultiNest**, required by `pymultinest`;
- A local installation or copy of the **ANDES ETC v3.2** interface used by `ETC.Custom_ETC`;
- Cached **ANDES ETC v5.2** CSV outputs for each target and spectral band;
- `petitRADTRANS` opacity files for the relevant molecules;
- The `petitRADTRANS` input-data path correctly configured for your system.

```text

├── ETC/

├── ETC_outputs_online/
│   ├── TRAPPIST_1_d/

│   │   ├── ... RIZ ... .csv
│   │   └── ... YJH ... .csv
│   ├── TRAPPIST_1_e/
│   │   ├── ... UBV ... .csv
│   │   ├── ... RIZ ... .csv
│   │   └── ... YJH ... .csv
│   └── ...
├── kernels/
│   ├── H2O_kernel.npy
│   ├── O2_kernel.npy
│   ├── CO2_kernel.npy
│   └── CH4_kernel.npy
├── results/
│   ├── planet_name/
│   │   ├── v_tot_planet_name.txt
│   │   ├── 1/
│   │   │   ├── CC1d_H2O_0.npy
│   │   │   ├── CC1d_H2O_1.npy
│   │   │   └── ...
│   │   ├── 2/
│   │   └── ...
│   └── ...
└── multinest_outputs/
    └── ... Bayesian-analysis outputs ...
```

The `simulation.py` script creates the `results/` directory and saves the CCFs used by `bayesian_analysis.py`. The Bayesian script then reads those CCFs, loads the corresponding molecular kernels from `kernels/`, and writes its MultiNest outputs and Jeffreys-scale summaries to `multinest_outputs/`.

## Usage notes

`simulation.py` is designed to be called with one target planet at a time. It expects 12 command-line arguments after the script name:

```bash
python simulation.py \
  planet_name \
  semi_major_axis_au \
  inclination_deg \
  orbital_period_days \
  transit_duration_hr \
  equilibrium_temperature_K \
  planet_radius_Rearth \
  stellar_radius_Rsun \
  surface_gravity_m_s2 \
  stellar_spectral_type \
  J_magnitude_Vega \
  v_tot_km_s
```

For example:

```bash
python simulation.py "TRAPPIST-1 e" 0.029 89.7 6.101 0.929 229 0.920 0.119 7.1 M8V 11.35 -51.78
```

After the CCFs have been generated, the Bayesian analysis can be run with:

```bash
python bayesian_analysis.py
```

The Bayesian script automatically searches the `results/` directory for available planets, exposure labels, molecules, and CCF files.

The six exposure labels used in the current implementation correspond to cumulative observing times equivalent to:

```text
1 → 3 transits
2 → 10 transits
3 → 25 transits
4 → 50 transits
5 → 75 transits
6 → 100 transits
```

The public scripts generate and analyze the core CCF and Bayesian-evidence products, but they do not automatically reproduce every table, figure, target-selection step, convergence diagnostic, or comparison run shown in the paper.
